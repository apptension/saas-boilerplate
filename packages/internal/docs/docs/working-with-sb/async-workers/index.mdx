# Background Jobs

<ProjectName /> supports background job processing for tasks that shouldn't block the request/response cycle.

---

## Two Approaches

| Environment | Technology | Use Case |
|-------------|------------|----------|
| **Local Development** | Celery + Redis | Immediate feedback, debugging |
| **Production (AWS)** | AWS Lambda | Serverless, auto-scaling |

---

## How It Works

### Local (Celery)

```
Django Request → Celery Task → Redis Queue → Celery Worker → Task Execution
```

### Production (Lambda)

```
Django Request → EventBridge → Lambda Function → Task Execution
```

## Quick Example

Creating a simple background task:

```python title="packages/backend/apps/myapp/tasks.py"
from celery import shared_task

@shared_task
def send_welcome_email(user_id: int):
    # This runs in the background
    user = User.objects.get(id=user_id)
    send_email(user.email, "Welcome!")
```

Triggering the task:

```python
# This returns immediately
send_welcome_email.delay(user.id)
```

## Local Development

Workers run automatically with `pnpm saas up`. To start them manually:

```bash
# Start Celery worker
cd packages/backend
pdm run celery -A config worker -l info

# Start Celery beat (scheduler)
pdm run celery -A config beat -l info
```
