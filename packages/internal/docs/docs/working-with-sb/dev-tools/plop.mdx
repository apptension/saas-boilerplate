The `plop` tool is a powerful micro-generator framework that automates the creation of project files and ensures consistency across the codebase. We've configured comprehensive generators optimized for this project's architecture and design patterns, supporting everything from simple components to full-stack CRUD modules.

## Quick Start

Run the plop generator from the `packages/webapp` directory:

```bash
cd packages/webapp
pnpm plop
# or
pnpm generate
```

You'll be presented with an interactive menu to select a generator.

## Available Generators

### Frontend Generators

| Generator | Description |
|-----------|-------------|
| `component` | React component with tests, Storybook stories, and optional GraphQL/hook files |
| `hook` | React hook with tests (supports state, effect, query, mutation, custom types) |
| `page` | Page/route with navigation, skeleton loading, and optional data fetching |
| `form` | Multi-field form with validation, types, and optional GraphQL mutation |
| `modal` | Modal/dialog component (confirm, form, info, or custom variants) |
| `table` | Data table with pagination, search, row actions, and skeleton |
| `context` | React Context provider with typed hook (state, reducer, or store patterns) |

### Backend Generators

| Generator | Description |
|-----------|-------------|
| `backend` | Django app with model, serializers, GraphQL schema, admin, and tests |

### Full-Stack Generators

| Generator | Description |
|-----------|-------------|
| `crud` | Complete CRUD module with frontend components and optional backend API |

### Utility Generators

| Generator | Description |
|-----------|-------------|
| `icon` | Register a custom SVG icon |
| `notification` | Notification component |
| `email` | Email template using @react-email |

## Component Generator

Creates a new React component with:
- Component file with TypeScript types
- Index file for clean exports
- Storybook stories (CSF3 format)
- Test file with basic specs
- Optional GraphQL operations file
- Optional custom hook file

### Usage

```bash
pnpm plop component
```

### Prompts

1. **Component name** - e.g., `UserProfile`, `PaymentCard`
2. **Directory** - Select where to create the component
3. **Has props?** - Include TypeScript props interface
4. **Include GraphQL?** - Add a GraphQL operations file
5. **Include hook?** - Add a custom hook file

### Generated Structure

```
src/[directory]/userProfile/
├── index.ts
├── userProfile.component.tsx
├── userProfile.stories.tsx
├── userProfile.graphql.ts (optional)
├── userProfile.hook.ts (optional)
└── __tests__/
    └── userProfile.component.spec.tsx
```

## Backend Generator

Creates a Django app with:
- Django model with fields
- DRF serializers
- GraphQL schema (types, queries, mutations)
- Django admin registration
- Factory Boy factories for testing
- GraphQL schema tests

### Usage

```bash
pnpm plop backend
```

### Prompts

1. **App/model name** - e.g., `Product`, `BlogPost`
2. **Plural name** - Auto-generated or custom
3. **Template** - Choose from entity templates or start custom
4. **Tenant-dependent?** - Multi-tenancy support
5. **Register in admin?** - Django admin registration
6. **Generate tests?** - Factory and schema tests

### Generated Structure

```
packages/backend/apps/product/
├── __init__.py
├── apps.py
├── models.py
├── serializers.py
├── schema.py
├── admin.py (optional)
└── tests/
    ├── __init__.py
    ├── factories.py
    └── test_schema.py
```

### After Generation

1. Add app to `INSTALLED_APPS` in settings
2. Add schema to root GraphQL config
3. Run migrations: `pnpm saas backend makemigrations && pnpm saas backend migrate`
4. Download schema: `pnpm saas webapp graphql download-schema`


## Best Practices

### Naming Conventions

- **Components**: PascalCase (e.g., `UserProfile`)
- **Hooks**: camelCase with `use` prefix (e.g., `useFormValidation`)
- **Files**: camelCase (e.g., `userProfile.component.tsx`)
- **Directories**: camelCase (e.g., `userProfile/`)
- **Django apps**: snake_case (e.g., `user_profile/`)

### After Generation

1. **Review generated code** - Templates provide a starting point
2. **Update GraphQL operations** - Customize queries/mutations for your schema
3. **Run type-check**: `pnpm type-check`
4. **Run tests**: `pnpm test`
5. **Update snapshots if needed**: `pnpm test --updateSnapshot`

### Internationalization

All generated text uses `react-intl` with the pattern:
```tsx
<FormattedMessage
  defaultMessage="Your text here"
  id="ComponentName / Description"
/>
```

## Troubleshooting

### "Command not found: plop"

Ensure you're in the `packages/webapp` directory and dependencies are installed:
```bash
cd packages/webapp
pnpm install
```

### Template errors

If you encounter Handlebars errors, check that:
1. The template files exist in `plop/[generator]/templates/`
2. Handlebars syntax is correct (use `\{{` to escape if needed)

### File already exists

Plop won't overwrite existing files. Delete the existing files first or choose a different name.

### Full-stack mode not generating backend

Make sure you selected "Full-stack" mode when prompted. Backend files are only generated in full-stack mode.

### GraphQL types not available

After generating code with GraphQL operations:
1. Ensure your backend is running
2. Download the schema: `pnpm saas webapp graphql download-schema`
3. Generate types: `pnpm nx run webapp-api-client:graphql-codegen`
