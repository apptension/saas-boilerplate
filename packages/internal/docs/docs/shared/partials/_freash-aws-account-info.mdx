:::caution New AWS Accounts Have Limitations

Freshly created AWS accounts have service limits that may affect deployment. These limits typically increase automatically with account age and usage.

**Common limitations:**

- **AWS SES Sandbox Mode:** Can only send emails to verified addresses. [Request production access](https://docs.aws.amazon.com/ses/latest/dg/request-production-access.html) for unrestricted email sending.

- **CodeBuild Concurrent Build Limit:** Limited to 1 concurrent build, which slows CI/CD pipelines. Set `SB_CI_MODE` to `simple` as a workaround, or [request a limit increase](https://docs.aws.amazon.com/servicequotas/latest/userguide/request-quota-increase.html).

- **CloudFront Distribution Creation:** May require account verification before you can create CloudFront distributions.

- **ECS/Fargate Limits:** May have low task limits initially.

**Recommendations:**

1. Request limit increases proactively through AWS Service Quotas
2. Use `SB_CI_MODE=simple` until CodeBuild limits are increased
3. Start with a single environment (qa) before adding more
4. See [Troubleshooting](/aws/troubleshooting) for solutions to common issues

:::
