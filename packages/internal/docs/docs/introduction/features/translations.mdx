# Translations & Internationalization

<ProjectName /> includes a comprehensive i18n system that goes beyond static translation files. Translations can be managed through Django Admin, published to cloud storage (S3/R2/B2), and automatically generated using OpenAI.

## Key Features

| Feature | Description |
|---------|-------------|
| **Dynamic Translations** | Update translations via admin without redeploying |
| **Cloud Publishing** | Translations served from S3/CloudFront/R2 for optimal performance |
| **AI Translation** | Automatically translate using OpenAI with context awareness |
| **Build-Time Sync** | Extract keys from source code and sync to database |
| **Version Control** | Rollback to previous translation versions |
| **RTL Support** | Full right-to-left language support (Arabic, Hebrew) |

## Build-Time Key Extraction

Translation keys are extracted from source code using `formatjs`:

```tsx
// Source code with FormattedMessage
<FormattedMessage 
  defaultMessage="Welcome back, {name}!" 
  id="Dashboard / Welcome message"
/>
```

```bash
# Extract all keys to master.json
cd packages/webapp
pnpm extract-intl:master

# Sync keys to database (uses docker exec)
pnpm translations:sync

# Or combine both steps
pnpm translations:extract-and-sync
```

The extraction covers all webapp packages:
- `webapp/src/**/*.tsx`
- `webapp-libs/webapp-core/src/**/*.tsx`
- `webapp-libs/webapp-tenants/src/**/*.tsx`
- `webapp-libs/webapp-management-dashboard/src/**/*.tsx`
- ... and all other webapp-libs

### Deployment Bundle

For deployments, a `translations_export.json` bundle is generated:

```javascript
// packages/webapp/scripts/exportTranslations.js
const exportData = {
  locales: data.locales,      // All active locales with metadata
  translations: data.translations,  // All translations per locale
};
fs.writeFileSync('translations_export.json', JSON.stringify(exportData));
```

This bundle enables:
- Pre-populating databases in new environments
- CI/CD pipelines without database access
- Backup/restore of translation data

## Cloud Storage Publishing

Translations are published to cloud storage for optimal frontend loading:

```python
# packages/backend/apps/translations/services.py
class TranslationPublisher:
    def publish(self, locale: Locale, user) -> TranslationVersion:
        # Generate translation JSON
        translations = self.generate_translation_bundle(locale)
        version_hash = self.generate_version_hash(translations)
        
        # Upload to storage (S3, R2, B2, or local)
        versioned_path = f"{locale.code}/{version_string}.json"
        current_path = f"{locale.code}.json"
        
        self._upload_json(versioned_path, translations)  # For history
        self._upload_json(current_path, translations)    # Current version
        
        # Create version record for rollback
        version = TranslationVersion.objects.create(
            locale=locale,
            version=version_string,
            s3_key=versioned_path,
            content_hash=version_hash,
        )
        return version
```

### Storage Backend Support

Uses the same storage backend as the rest of the application:

| Backend | Configuration | Use Case |
|---------|--------------|----------|
| **AWS S3** | `STORAGE_BACKEND=s3` | Production with CloudFront CDN |
| **Cloudflare R2** | `STORAGE_BACKEND=r2` | Cost-effective with global CDN |
| **Backblaze B2** | `STORAGE_BACKEND=b2` | Budget-friendly option |
| **Local** | `STORAGE_BACKEND=local` | Development only |

## AI-Powered Translation

Automatically translate your app using OpenAI:

```python
# packages/backend/apps/translations/ai_service.py
class AITranslationService:
    def translate_batch(
        self,
        keys_with_text: List[Dict[str, str]],
        source_locale: str,
        target_locale: str,
        app_context: str = "a modern SaaS web application",
    ) -> List[TranslationResult]:
        # Build context-aware prompt
        prompt = self._build_translation_prompt(keys_with_text, ...)
        
        response = self.client.chat.completions.create(
            model=self.model,  # gpt-4 default
            messages=[
                {"role": "system", "content": self._get_system_prompt(...)},
                {"role": "user", "content": prompt},
            ],
            temperature=0.3,  # Low for consistent translations
            response_format={"type": "json_object"},
        )
        # ...
```

### AI Translation Features

- **Context Awareness**: Message ID and description passed to AI
- **Placeholder Preservation**: `{name}`, `{count}` kept intact
- **Batch Processing**: Efficient API usage with configurable batch sizes
- **Progress Tracking**: Real-time progress via admin UI
- **Browser Polling**: No Celery required - works with any deployment
- **Error Recovery**: Failed keys tracked for retry

## Data Models

### TranslationKey

Represents a unique translation key extracted from source code:

```python
class TranslationKey(models.Model):
    key = models.CharField(max_length=512, unique=True)  # Message ID
    default_message = models.TextField()  # English default
    description = models.TextField(blank=True)  # Context for translators
    is_deprecated = models.BooleanField(default=False)  # Key removed from code
    first_seen_at = models.DateTimeField(auto_now_add=True)
    last_seen_at = models.DateTimeField(auto_now=True)
```

### Locale

Supported language/locale configuration:

```python
class Locale(models.Model):
    code = models.CharField(max_length=10)  # 'en', 'pl', 'en-US'
    name = models.CharField(max_length=100)  # 'English', 'Polish'
    native_name = models.CharField(max_length=100)  # 'English', 'Polski'
    is_default = models.BooleanField(default=False)
    is_active = models.BooleanField(default=True)
    rtl = models.BooleanField(default=False)  # Right-to-left
    sort_order = models.PositiveIntegerField(default=0)
```

### Translation

Individual translation for a key in a locale:

```python
class Translation(models.Model):
    class Status(models.TextChoices):
        DRAFT = 'draft'
        REVIEW = 'review'
        APPROVED = 'approved'
        PUBLISHED = 'published'

    key = models.ForeignKey(TranslationKey, on_delete=models.CASCADE)
    locale = models.ForeignKey(Locale, on_delete=models.CASCADE)
    value = models.TextField()
    status = models.CharField(choices=Status.choices, default=Status.DRAFT)
    translated_by = models.ForeignKey(User, null=True)
```

## Best Practices

### Message ID Convention

```
"Section / Description"
```

Examples:
- `"Dashboard / Welcome message"`
- `"Profile / Save button"`
- `"Auth / Login error"`

### Placeholders

```tsx
// вң… Good - uses named placeholders
<FormattedMessage 
  defaultMessage="You have {count} notifications" 
  values={{ count: 5 }}
/>

// вқҢ Bad - no placeholder support
<span>You have {count} notifications</span>
```

### Don't Include Emojis

```tsx
// вқҢ Bad - emojis in translations
<FormattedMessage defaultMessage="рҹҺү Success!" />

// вң… Good - use toast variants for visual feedback
toast({ description: 'Success!', variant: 'success' });
```
