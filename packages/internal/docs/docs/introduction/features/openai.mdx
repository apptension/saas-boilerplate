## Quick Start

### 1. Set Up Environment Variables

Add your OpenAI API key to your environment configuration:

```bash title=".env"
OPENAI_API_KEY=sk-your-api-key-here
OPENAI_MODEL=gpt-4  # Optional, defaults to gpt-4
```

| Variable | Required | Default | Description |
|----------|----------|---------|-------------|
| `OPENAI_API_KEY` | Yes | - | Your OpenAI API key from [platform.openai.com](https://platform.openai.com/api-keys) |
| `OPENAI_MODEL` | No | `gpt-4` | The OpenAI model to use for generation |

### 2. Access the Feature

Once configured, the SaaS Ideas Generator is available at:

```
/en/{tenantId}/saas-ideas
```

Users can enter comma-separated keywords, and the AI generates creative SaaS product ideas with structured output including product name, description, target market, key features, and monetization strategy.

## Supported Models

The integration supports multiple OpenAI models with automatic fallback:

| Model | Description | Notes |
|-------|-------------|-------|
| `gpt-4` | Default, most capable | Requires GPT-4 API access |
| `gpt-4-turbo-preview` | Faster GPT-4 variant | Fallback if `gpt-4` unavailable |
| `gpt-3.5-turbo` | Fast and cost-effective | Final fallback option |

The client automatically tries alternative models if the requested model is unavailable, ensuring resilience.

## GraphQL API

### Mutation: `generateSaasIdeas`

```graphql
mutation generateSaasIdeasMutation($input: GenerateSaasIdeasMutationInput!) {
  generateSaasIdeas(input: $input) {
    response
  }
}
```

**Input:**

```graphql
input GenerateSaasIdeasMutationInput {
  keywords: [String]      # Array of keywords (e.g., ["CRM", "automation"])
  clientMutationId: String
}
```

**Response:**

```graphql
type GenerateSaasIdeasMutationPayload {
  response: String        # Generated SaaS ideas as formatted text
  clientMutationId: String
}
```

### Rate Limiting

:::caution Rate Limit
The `generateSaasIdeas` mutation is rate-limited to **3 requests per minute** per IP address to prevent abuse and manage API costs.
:::

## Frontend Implementation

The frontend component is located in `packages/webapp-libs/webapp-generative-ai/` and provides a chat-style interface.

### Using the Mutation

```tsx title="Example: Using the generateSaasIdeas mutation"
import { useMutation } from '@apollo/client';
import { gql } from '@sb/webapp-api-client/graphql';

const generateSaasIdeasMutation = gql(/* GraphQL */ `
  mutation generateSaasIdeasMutation($input: GenerateSaasIdeasMutationInput!) {
    generateSaasIdeas(input: $input) {
      response
    }
  }
`);

// In your component:
const [generateIdeas, { loading, data }] = useMutation(generateSaasIdeasMutation, {
  onCompleted: (data) => {
    const ideas = data?.generateSaasIdeas?.response;
    // Handle the generated ideas
  },
  onError: (error) => {
    // Handle errors (rate limiting, API issues, etc.)
  },
});

// Trigger the mutation:
await generateIdeas({
  variables: {
    input: {
      keywords: ['project management', 'AI', 'automation'],
    },
  },
});
```

### OpenAI Client

The OpenAI client (`packages/backend/apps/integrations/openai/client.py`) handles API communication:

```python title="Key features of the OpenAI client"
class OpenAIClient:
    # Singleton pattern for client reuse
    _client = None

    @classmethod
    def _get_client(cls):
        """Get or create OpenAI client instance."""
        if cls._client is None:
            if not settings.OPENAI_API_KEY:
                raise OpenAIClientException("OpenAI API key is not configured")
            cls._client = OpenAI(api_key=settings.OPENAI_API_KEY)
```

### GraphQL Mutation Handler

```python title="packages/backend/apps/integrations/schema.py"
class GenerateSaasIdeasMutation(graphene.relay.ClientIDMutation):
    class Input:
        keywords = graphene.List(graphene.String)

    response = graphene.String()

    @classmethod
    @ratelimit(key="ip", rate='3/min')  # Rate limiting
    def mutate_and_get_payload(cls, root, info, keywords):
        response_text = client.OpenAIClient.get_saas_ideas(keywords)
        return cls(response=response_text)
```

## Extending the Integration

### Adding a New AI Feature

To add your own AI-powered feature:

<Tabs>
<TabItem value="backend" label="1. Backend" default>

Create a new method in the OpenAI client:

```python title="packages/backend/apps/integrations/openai/client.py"
@staticmethod
def your_new_feature(input_data: str) -> str:
    """Your custom AI feature."""
    client = OpenAIClient._get_client()
    
    result = client.chat.completions.create(
        model=settings.OPENAI_MODEL,
        messages=[
            {"role": "system", "content": "Your system prompt here"},
            {"role": "user", "content": input_data}
        ],
        temperature=0.7,
        max_tokens=1000,
    )
    
    return result.choices[0].message.content.strip()
```

Add the GraphQL mutation:

```python title="packages/backend/apps/integrations/schema.py"
class YourNewMutation(graphene.relay.ClientIDMutation):
    class Input:
        input_data = graphene.String(required=True)

    response = graphene.String()

    @classmethod
    @ratelimit(key="ip", rate='10/min')
    def mutate_and_get_payload(cls, root, info, input_data):
        response = client.OpenAIClient.your_new_feature(input_data)
        return cls(response=response)


class Mutation(graphene.ObjectType):
    generate_saas_ideas = GenerateSaasIdeasMutation.Field()
```

## Error Handling

The integration handles several error scenarios:

| Error | Cause | User Message |
|-------|-------|--------------|
| Missing API Key | `OPENAI_API_KEY` not configured | "OpenAI API key is not configured" |
| Rate Limited | More than 3 requests/minute | "Rate limit exceeded." |
| API Error | OpenAI service issues | "OpenAI service is currently unavailable. Please try again in a couple seconds." |
| Model Unavailable | Requested model not accessible | Automatic fallback to alternative model |

## Best Practices

1. **Secure your API key**: Never commit `OPENAI_API_KEY` to version control. Use environment variables or secrets management.

2. **Monitor usage**: OpenAI API calls have costs. Implement appropriate rate limiting and monitoring for production.

3. **Handle errors gracefully**: Always provide meaningful feedback to users when AI operations fail.

4. **Test with mocks**: Use the provided test fixtures for unit testing without making actual API calls.

5. **Consider caching**: For repeated queries, consider caching responses to reduce API costs and latency.
