# AI Agent with Model Context Protocol

<ProjectName /> is **MCP-ready** - implementing the [Model Context Protocol](https://modelcontextprotocol.io/) to provide a production-grade AI assistant that can intelligently interact with your application data through natural language.

:::tip The Future of SaaS
MCP represents a paradigm shift in how AI integrates with applications. Instead of building custom integrations for each AI provider, MCP provides a standardized protocol that any AI model can use to discover and execute tools. This makes your SaaS future-proof - as new AI models emerge, they can immediately leverage your existing tools.
:::

## Why This Matters

Traditional AI integrations require:
- Custom API endpoints for each operation
- Manual prompt engineering for data access
- Rebuilding when switching AI providers
- Complex permission handling

With MCP, you get:
- **Standardized Tool Discovery**: AI models automatically discover what they can do
- **Provider Agnostic**: Works with OpenAI, Claude, and any MCP-compatible AI
- **RBAC Integration**: Tools respect your existing permission system
- **Type-Safe Operations**: GraphQL operations exposed as strongly-typed tools
- **Activity Logging**: All AI actions are tracked with full audit trail

### Full RBAC Integration

Every tool call is checked against the user's permissions:

```python
# Backend permission check on every tool call
TOOL_PERMISSIONS = {
    # View permissions
    "get_projects": "management.projects.view",
    "get_revenue_lines": "management.financial.view",
    "get_forecast_insights": "management.analytics.view",
    
    # Edit permissions for mutations
    "create_project": "management.projects.edit",
    "update_revenue_line": "management.financial.edit",
    "delete_invoice": "management.invoices.edit",
}

def check_tool_permission(user, tenant, tool_name):
    required = TOOL_PERMISSIONS.get(tool_name)
    if required and not user_has_permission(user, tenant, required):
        return False, f"Missing permission: {required}"
    return True, None
```

**Security Flow:**
1. User sends message → Backend validates tenant access
2. Backend filters available tools by user's permissions
3. AI only sees tools the user can actually use
4. Each tool execution is re-validated before execution
5. All actions logged with `AI_AGENT` actor type

### Real-Time Streaming via GraphQL Subscriptions

The chat interface uses GraphQL subscriptions over WebSockets for real-time communication:

```typescript
// GraphQL Subscription
subscription AiChat($conversationId: String!) {
  aiChat(conversationId: $conversationId) {
    event {
      eventType    // status, tool_start, tool_complete, content, done, error
      message      // Status messages
      toolName     // MCP tool being executed
      toolDisplayName  // Human-readable tool name
      success      // Tool execution success
      hasData      // Whether tool returned data
      text         // Response content (streamed word-by-word)
      toolsUsed    // Final list of tools used
      conversationId
    }
  }
}

// Mutation to send message
mutation SendAiMessage($message: String!, $tenantId: String!, $conversationId: String!, $history: [JSONString]) {
  sendAiMessage(message: $message, tenantId: $tenantId, conversationId: $conversationId, history: $history) {
    ok
    error
  }
}
```

**How it works:**
1. Frontend subscribes to `aiChat` subscription with unique `conversationId`
2. User sends message via `sendAiMessage` mutation
3. Backend processes asynchronously, broadcasting events via subscription
4. Frontend receives real-time updates: status → tool execution → streaming content → done

Users see:
- "Connecting to data sources..." → "Connected to 45 data sources"
- Real-time tool execution status with human-readable names
- Word-by-word streaming response with natural typing delays
- Clickable entity links in responses

## Infrastructure

### Apollo MCP Server

The MCP server is a containerized service that exposes GraphQL operations as MCP tools:

```yaml
# packages/infra/mcp-server/config.yaml
endpoint: http://backend:5001/api/graphql/

forward_headers:
  - Authorization
  - X-Tenant-ID
  - X-AI-Agent-Request

transport:
  type: streamable_http
  address: 0.0.0.0
  port: 4000

# Only allow pre-defined mutations (not arbitrary GraphQL)
overrides:
  mutation_mode: explicit
```

### Adding New MCP Tools

Create a `.graphql` file in `packages/infra/mcp-server/operations/`:

```graphql
# @tool(name: "get_custom_data", description: "Fetches custom data for analysis")
query GetCustomData($tenantId: ID!, $dateRange: DateRangeInput) {
  customData(tenantId: $tenantId, dateRange: $dateRange) {
    id
    name
    value
    timestamp
  }
}
```

The `@tool` directive comment tells the MCP server:
- Tool name exposed to AI
- Description for AI to understand when to use it

### Mutation Modes

| Mode | Description | Use Case |
|------|-------------|----------|
| `none` | Mutations disabled | Read-only deployments |
| `explicit` | Only pre-defined mutation tools | **Production (default)** |
| `all` | AI can build any mutation | Development/testing only |

## Frontend Integration

### AI Assistant Hook

```tsx
import { useAiAssistant } from '@sb/webapp-ai-assistant/hooks';

function ChatInterface() {
  const {
    messages,
    isLoading,
    sendMessage,
    streamingState,
    isConnected,
  } = useAiAssistant();

  return (
    <div>
      {/* Status indicator */}
      {streamingState.status && (
        <div className="text-muted">{streamingState.status}</div>
      )}
      
      {/* Active tools */}
      {streamingState.activeTools.map(tool => (
        <ToolBadge key={tool.name} tool={tool} />
      ))}
      
      {/* Messages */}
      {messages.map(msg => (
        <Message key={msg.id} {...msg} />
      ))}
      
      {/* Input */}
      <ChatInput onSend={sendMessage} disabled={isLoading} />
    </div>
  );
}
```

## Security Considerations

### Built-in Protections

| Protection | Implementation |
|------------|----------------|
| **Tenant Isolation** | tenantId injected server-side, cannot be manipulated |
| **RBAC Enforcement** | Every tool checked against user permissions |
| **Rate Limiting** | 30 MCP requests/min, 20 chat requests/min |
| **Mutation Control** | Default `explicit` mode - only pre-defined mutations |
| **Audit Trail** | All actions logged with AI_AGENT actor type |
| **Auth Forwarding** | Authorization headers forwarded to GraphQL |

### Best Practices

1. **Start with `mutation_mode: explicit`** - only enable `all` for development
2. **Review tool permissions** - ensure sensitive operations require appropriate permissions
3. **Monitor AI actions** - use Activity Logs to audit AI behavior
4. **Set appropriate rate limits** - prevent abuse while allowing productive use
