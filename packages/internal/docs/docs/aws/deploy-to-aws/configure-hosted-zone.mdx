# Step 2: Set Up Your Domain

This step configures DNS in AWS Route 53 so your application can be accessed via your custom domain (e.g., `app.yourcompany.com`).

**Estimated time: 10-30 minutes** (depending on your domain situation)

- ✅ Obtain the Hosted Zone ID and Name (needed for deployment)
- ✅ Understand how domains will be structured

## How <ProjectName /> Uses Your Domain

After deployment, your services will be available at subdomains:

| Service | URL Pattern | Example |
|---------|-------------|---------|
| Web Application | `app.[stage].[domain]` | `app.qa.example.com` |
| API | `api.[stage].[domain]` | `api.qa.example.com` |
| Admin Panel | `admin.[stage].[domain]` | `admin.qa.example.com` |
| CDN (Assets) | `cdn.[stage].[domain]` | `cdn.qa.example.com` |
| Documentation | `docs.[stage].[domain]` | `docs.qa.example.com` |
| Celery Flower | `flower.[stage].[domain]` | `flower.qa.example.com` |

For production, you can configure simpler URLs like `app.example.com` (without the stage prefix).

---

## What is a Hosted Zone?

A **hosted zone** is a container in Route 53 that holds DNS records for a domain. Think of it as the "address book" for your domain—it tells the internet where to find your services.

**Example:** If you own `example.com`, the hosted zone contains records that say:
- `app.qa.example.com` → points to your web app
- `api.qa.example.com` → points to your backend

### Register a New Domain with AWS

The easiest option—AWS handles everything:

1. Go to [Route 53 Console](https://console.aws.amazon.com/route53)
2. Click **Registered domains** → **Register domain**
3. Search for your desired domain and check availability
4. Follow the purchase flow (prices vary, typically $10-50/year for common TLDs)
5. Wait for registration to complete (usually a few minutes to hours)

**After registration:**
- AWS automatically creates a hosted zone
- You can find it in Route 53 → **Hosted zones**

</TabItem>
<TabItem value="external" label="I own a domain from another registrar">

### Use a Domain from GoDaddy, Namecheap, etc.

You'll create a hosted zone in AWS and point your domain to it.

#### Step 1: Create a Hosted Zone in AWS

1. Go to [Route 53 Console](https://console.aws.amazon.com/route53)
2. Click **Hosted zones** → **Create hosted zone**
3. Enter your domain name (e.g., `example.com`)
4. Select **Public hosted zone**
5. Click **Create hosted zone**

#### Step 2: Update Your Domain's Nameservers

After creating the hosted zone, you'll see 4 nameservers (NS records) like:
```
ns-123.awsdns-45.com
ns-678.awsdns-90.co.uk
ns-111.awsdns-22.net
ns-333.awsdns-44.org
```

1. Copy all 4 nameserver addresses
2. Log in to your domain registrar (GoDaddy, Namecheap, etc.)
3. Find the nameserver settings for your domain
4. Replace the existing nameservers with the AWS ones
5. Save changes

:::warning DNS Propagation
After changing nameservers, it can take 24-48 hours for changes to propagate globally. In practice, it's often much faster (minutes to hours).
:::

### Use a Subdomain of an Existing Domain

If you want to deploy to a subdomain (e.g., `saas.yourcompany.com`) without affecting the main domain:

1. Go to [Route 53 Console](https://console.aws.amazon.com/route53)
2. Click **Hosted zones** → **Create hosted zone**
3. Enter the subdomain (e.g., `saas.yourcompany.com`)
4. Select **Public hosted zone**
5. Click **Create hosted zone**

#### Delegate the Subdomain

In your parent domain's hosted zone:

1. Find the NS records for your new subdomain hosted zone
2. Go to the parent domain's hosted zone
3. Create a new NS record for the subdomain with those nameservers

**Example:** If creating `saas.yourcompany.com`, add NS records to the `yourcompany.com` hosted zone.

## Get Your Hosted Zone Details

After setting up your hosted zone, you need two values for deployment:

### Find in AWS Console

1. Go to [Route 53 → Hosted zones](https://console.aws.amazon.com/route53/v2/hostedzones)
2. Click on your hosted zone
3. Note down:

| Value | Where to Find It | Example |
|-------|------------------|---------|
| **Hosted Zone ID** | Shown in the hosted zone details | `Z0123456789ABCDEFGHIJ` |
| **Hosted Zone Name** | Your domain name | `example.com` |

:::tip Quick Access
You can also open the AWS Console directly:
```bash
pnpm saas aws login
```
:::

### Store for Later

You'll use these values in Step 3. Write them down:

```
Hosted Zone ID:   ____________________________
Hosted Zone Name: ____________________________
```

## Using External DNS (Advanced)

If you cannot or don't want to use Route 53 for DNS, you can use an external DNS provider. This requires more manual configuration:

1. **Skip** setting `SB_HOSTED_ZONE_ID` and `SB_HOSTED_ZONE_NAME`
2. Manually create SSL certificates in AWS Certificate Manager (ACM)
3. Set `SB_CLOUDFRONT_CERTIFICATE_ARN` and `SB_LOAD_BALANCER_CERTIFICATE_ARN`
4. After deployment, create CNAME records in your DNS provider pointing to CloudFront and the Load Balancer

:::warning More Complex
This approach requires understanding of SSL certificates, CloudFront, and DNS. We recommend using Route 53 unless you have specific requirements.
:::

## Troubleshooting

### Hosted Zone Not Showing Records

- Newly created hosted zones may take a few minutes to become active
- Refresh the page or wait 5 minutes

### Domain Not Resolving After Nameserver Change

- DNS propagation can take up to 48 hours
- Check propagation status at [whatsmydns.net](https://www.whatsmydns.net/)
- Clear your local DNS cache: `sudo dscacheutil -flushcache` (macOS)

### DNSSEC Errors

If your domain registrar has DNSSEC enabled, you may need to disable it before changing nameservers, then re-enable after pointing to AWS.
