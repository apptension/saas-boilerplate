## Time & Cost Expectations

| Aspect | Estimate |
|--------|----------|
| **First Deployment Time** | 2-4 hours (including AWS setup) |
| **Subsequent Deployments** | 15-30 minutes via CI/CD |
| **AWS Monthly Cost** | ~$50-150/month for staging, ~$200-500+ for production |
| **Required AWS Services** | ECS Fargate, RDS, ElastiCache, S3, CloudFront, Route 53 |

:::tip Cost Optimization
Start with a staging environment to learn the deployment process. You can delete resources when not in use to minimize costs during development.
:::

---

## AWS Services Used

The boilerplate leverages these AWS services:

| Service | Purpose |
|---------|---------|
| **ECS Fargate** | Container orchestration for backend API and Celery workers |
| **RDS PostgreSQL** | Managed relational database |
| **ElastiCache Redis** | Caching, session storage, and Celery message broker |
| **S3** | Static file hosting, asset storage, and CI/CD artifacts |
| **CloudFront** | CDN for frontend assets with edge caching |
| **Route 53** | DNS management and domain routing |
| **Secrets Manager** | Secure storage for API keys and credentials |
| **Systems Manager** | Environment variable management (Parameter Store) |
| **CloudWatch** | Logging, monitoring, and alerting |
| **CodePipeline** | CI/CD orchestration |
| **CodeBuild** | Build and deploy automation |
| **ACM** | SSL/TLS certificate management |

## Before You Begin

Make sure you have completed the [local development setup](/getting-started) and can run the application locally. This ensures you understand the project structure before deploying.

### Required Knowledge

You don't need to be an AWS expert, but familiarity with these concepts helps:

- **Basic command-line usage** – Running terminal commands
- **Environment variables** – What they are and how they work
- **DNS basics** – Understanding domains and subdomains
- **Git** – Pushing code to repositories

:::info New to AWS?
Don't worry! We'll explain each step. The guide includes links to AWS documentation when you need more context.
:::

## Quick Start Checklist

Before starting deployment, ensure you have:

- [ ] Working local development environment ([Quick Start](/getting-started))
- [ ] AWS account (or access to one)
- [ ] Domain name you control (can be purchased through AWS Route 53)
- [ ] ~2-4 hours of uninterrupted time for first deployment
- [ ] Docker Hub account (free tier is fine)
