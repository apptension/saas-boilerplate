## AWS Services Used

The boilerplate leverages these AWS services:

| Service | Purpose |
|---------|---------|
| **ECS Fargate** | Container orchestration for backend and workers |
| **RDS PostgreSQL** | Managed database for persistent data |
| **ElastiCache Redis** | Caching and session storage |
| **S3** | Static file hosting and asset storage |
| **CloudFront** | CDN for frontend assets and API caching |
| **Route 53** | DNS management and domain routing |
| **Secrets Manager** | Secure storage for credentials |
| **CloudWatch** | Logging, monitoring, and alerts |
| **CodePipeline** | CI/CD automation |

## Deployment Checklist

Before deploying to production, ensure you have:

- [ ] AWS account with appropriate permissions
- [ ] Configured AWS CLI credentials
- [ ] Set up a hosted zone in Route 53
- [ ] Created environment-specific configuration
- [ ] Reviewed security settings
- [ ] Set up monitoring and alerting

:::caution Fresh AWS Account
If you're using a new AWS account, you may need to request service quota increases for some services. See our [troubleshooting guide](/aws/troubleshooting) for details.
:::
