## Key Components

| Component | AWS Service | Purpose |
|-----------|-------------|---------|
| **Web Application** | S3 + CloudFront | Hosts React frontend, globally cached |
| **Backend API** | ECS Fargate + ALB | Django REST/GraphQL API |
| **Admin Panel** | ECS Fargate + ALB | Django Admin interface |
| **Background Jobs** | ECS Fargate (Celery) | Async task processing |
| **Scheduled Tasks** | ECS Fargate (Celery Beat) | Periodic job scheduling |
| **Serverless Functions** | Lambda | Short-lived async tasks |
| **Database** | RDS PostgreSQL | Persistent data storage |
| **Cache/Broker** | ElastiCache Redis | Caching + Celery message broker |
| **Static Assets** | S3 | File uploads, static files |
| **CDN** | CloudFront | Global content delivery |
| **DNS** | Route 53 | Domain name resolution |
| **SSL/TLS** | ACM | Certificate management |
| **Secrets** | Secrets Manager + Parameter Store | Secure configuration |

## Infrastructure as Code

All infrastructure is defined in TypeScript using AWS CDK:

```
packages/
├── infra/
│   └── infra-shared/
│       └── src/
│           └── stacks/
│               ├── global/     # ECR, shared resources
│               ├── main/       # VPC, ALB, ECS Cluster
│               ├── db/         # RDS, ElastiCache
│               ├── ci/         # CodePipeline, CodeBuild
│               └── components/ # CloudFront, S3
├── webapp/
│   └── infra/                  # Webapp-specific CDK
├── backend/
│   └── infra/                  # Backend-specific CDK
└── workers/
    └── (serverless.yml)        # Lambda functions
```
