## Step 3: Start the Application {#start-the-app}

Now let's start everything and verify it's working correctly.

<WSLNotice />

### Option A: Start Everything (Recommended)

The easiest way to start all services is with a single command:

```bash
pnpm saas up
```

This command will:
1. Build email templates
2. Start all backend services (database, Redis, API, workers)
3. Wait for the backend to be ready
4. Start the webapp development server

:::info First Run
The first run takes longer (5-10 minutes) as Docker downloads and builds images. Subsequent starts are much faster (under 1 minute).
:::

### Option B: Start Services Separately

If you prefer more control, start services individually:

```bash
# Start backend services only
pnpm nx run core:docker-compose:up

# In a new terminal, start the webapp
pnpm nx start webapp
```

## Step 4: Verify Everything is Working

Once startup completes, you should be able to access:

| Service | URL | What to Expect |
|---------|-----|----------------|
| **Web App** | [http://localhost:3000](http://localhost:3000) | Login/signup page |
| **Backend API** | [http://localhost:5001](http://localhost:5001) | API root or GraphQL playground |
| **Admin Panel** | [http://admin.localhost:5001](http://admin.localhost:5001) | Django admin login |
| **Mailcatcher** | [http://localhost:1080](http://localhost:1080) | Email inbox (catches all outgoing emails) |
| **Workers** | [http://localhost:3005](http://localhost:3005) | Workers trigger server |

### Test the Setup

1. **Open the web app** at [http://localhost:3000](http://localhost:3000)
2. **Create an account** using the signup form
3. **Check Mailcatcher** at [http://localhost:1080](http://localhost:1080) for the verification email
4. **Verify your email** by clicking the link in Mailcatcher
5. **Log in** with your new account

:::tip Admin Access
You can also log in to the Admin Panel at [http://admin.localhost:5001](http://admin.localhost:5001) using the credentials you set in `ADMIN_EMAIL` and `ADMIN_DEFAULT_PASSWORD`.
:::

## Troubleshooting

### Docker Issues

<details>
<summary><strong>Error: Cannot connect to Docker daemon</strong></summary>

Make sure Docker Desktop is running. On macOS/Windows, look for the Docker icon in your system tray.

```bash
# Check if Docker is running
docker info
```

If Docker isn't running, start Docker Desktop and try again.

</details>

<details>
<summary><strong>Error: Port already in use</strong></summary>

Another application is using a required port. Find and stop it:

```bash
# Find what's using port 5001
lsof -i :5001

# Or on Windows
netstat -ano | findstr :5001
```

Common conflicts:
- Port 5432: Another PostgreSQL instance
- Port 6379: Another Redis instance
- Port 3000: Another development server

</details>

<details>
<summary><strong>Docker build fails or takes forever</strong></summary>

Try cleaning up Docker resources:

```bash
# Remove unused containers, networks, and images
docker system prune -a

# Rebuild from scratch
pnpm nx run core:docker-compose:up --build
```

</details>

### Backend Issues

<details>
<summary><strong>Database migration errors</strong></summary>

Reset the database and run migrations:

```bash
# Stop everything
pnpm saas down

# Remove the database volume
docker volume rm $(docker volume ls -q | grep db-data)

# Start fresh
pnpm saas up
```

</details>

<details>
<summary><strong>Backend not responding on localhost:5001</strong></summary>

Check if the backend container is running:

```bash
docker ps | grep backend
```

View backend logs for errors:

```bash
docker logs $(docker ps -qf "name=backend") --tail 100
```

</details>

### Frontend Issues

<details>
<summary><strong>Webapp shows blank page or errors</strong></summary>

1. Clear your browser cache or try incognito mode
2. Check the browser console for errors (F12)
3. Restart the webapp:

```bash
# Stop webapp (Ctrl+C) and restart
pnpm nx start webapp
```

</details>

<details>
<summary><strong>GraphQL errors or "Failed to fetch"</strong></summary>

The backend might not be ready yet. Wait for all services to be healthy:

```bash
# Check service health
docker ps
```

All containers should show "healthy" status. If the backend shows "starting", wait a moment.

</details>
