# Start a New Project

This guide walks you through creating a fresh <ProjectName/> project from scratch. By the end, you'll have a fully functional SaaS application running locally.

:::tip Estimated Time
**10-15 minutes** for first-time setup (depending on your internet speed for Docker image downloads).
:::

Already have a project? See [Join an Existing Project](./run-existing-project) instead.

## Step 1: Create Your Project

The fastest way to get started is with our CLI tool. Open your terminal and run:

<Tabs>
  <TabItem value="pnpm" label="pnpm" default>
    <CodeBlock>
      pnpm create saas-boilerplate my-saas-app
    </CodeBlock>
  </TabItem>
  <TabItem value="npm" label="npm">
    <CodeBlock>
      npm init saas-boilerplate my-saas-app
    </CodeBlock>
  </TabItem>
  <TabItem value="yarn" label="yarn">
    <CodeBlock>
      yarn create saas-boilerplate my-saas-app
    </CodeBlock>
  </TabItem>
</Tabs>

Replace `my-saas-app` with your desired project name.

:::info What this does
The CLI will:
1. Clone the SaaS Boilerplate repository
2. Install all dependencies
3. Create default environment files
4. Set up your local database
5. Guide you through initial configuration
:::

:::caution
The target directory must be **empty** or non-existent. The CLI will create it for you.
:::

Once complete, navigate into your project:

```bash
cd my-saas-app
```

Then skip to [Step 3: Start the Application](#start-the-app).

### Configure Environment Variables

The project uses `.env` files for configuration. Default values are provided in `.env.shared` files, but you need to customize a few settings.

:::caution Security
Never commit `.env` files to Git! Only `.env.shared` files (with non-secret defaults) should be committed.
:::

#### Root Configuration (`.env.shared`)

| Variable | Example | Description |
|----------|---------|-------------|
| `PROJECT_NAME` | `myapp` | Short project identifier (3-5 chars recommended to avoid AWS name length limits) |

#### Backend Configuration (`packages/backend/.env.shared`)

| Variable | Example | Description |
|----------|---------|-------------|
| `DJANGO_SECRET_KEY` | `Zs639zRcb5!9om2@tW2H6XG#Znj^TB^I` | **Required.** 50-character random string for cryptographic signing |
| `HASHID_FIELD_SALT` | `t5$^r*xsMRXn1xjzhRSl8I5Hb3BUW$4U` | **Required.** 50-character random string for ID hashing |
| `ADMIN_EMAIL` | `admin@example.com` | Email for the initial superadmin account |
| `ADMIN_DEFAULT_PASSWORD` | `AvPZpabgj9Z8` | Password for the initial superadmin account |

:::tip Generate Secret Keys
You can generate secure random strings using Python:

```bash
python3 -c "import secrets; print(secrets.token_urlsafe(50)[:50])"
```

Or using Node.js:

```bash
node -e "console.log(require('crypto').randomBytes(50).toString('base64').slice(0,50))"
```

Or use an online generator like [RandomKeygen](https://randomkeygen.com/).
:::
