# Join an Existing Project

This guide helps you set up your local development environment to work on an existing <ProjectName/> project that your team has already configured.

:::tip Before You Begin
Make sure you have:
- Access to the project's Git repository
- Any team-specific environment variables or secrets from your team lead
:::

Starting fresh? See [Start a New Project](./run-new-project) instead.

---

## Step 1: Clone the Repository

Get the project code from your team's repository:

```bash
git clone <your-project-repository-url>
cd <project-name>
```

## Step 2: Get Environment Files

Your team should provide you with the necessary `.env` files. These contain configuration specific to your project.

:::caution Don't Commit Secrets
Never commit `.env` files to Git. They should be shared securely (password manager, encrypted file sharing, etc.).
:::

You'll need `.env` files in these locations:
- Root directory (`.env`)
- `packages/backend/.env`
- `packages/workers/.env` (if using workers)

**If your team hasn't provided these**, you can create them from the `.env.shared` templates:

```bash
# Copy shared env files as a starting point
cp .env.shared .env
cp packages/backend/.env.shared packages/backend/.env
```

Then ask your team for any project-specific values you need to update.
